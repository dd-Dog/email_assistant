# 公司配置目录

## 📋 功能说明

本目录存储公司整体信息，用于构建"公司知识图谱"，让AI全面理解公司业务、组织架构和核心能力。

## 📂 文件结构

```
config/公司/
├── 公司信息.xlsx      ← 公司简介、业务范围、核心竞争力
├── 部门信息.xlsx      ← 部门职能、协作关系、KPI指标
└── README.md          ← 本说明文档
```

## 💡 使用说明

### 1. 首次配置
```bash
# 1. 复制模板
python create_company_excel_template.py

# 2. 打开Excel文件
config/公司/公司信息.xlsx
config/公司/部门信息.xlsx

# 3. 根据实际情况填写
按照表头提示填写公司和部门信息
```

### 2. AI如何使用这些信息

**场景1：客户需求分析**
```
客户：我们需要增加WiFi 6功能

AI分析：
1. 查询公司业务领域 → 有"硬件设计"能力
2. 查询协作部门 → 需要研发部（硬件）+ 采购部（WiFi模块）
3. 查询相关产品线 → G20项目使用类似技术栈
4. 给出建议：技术可行，需要研发部评估，采购部确认WiFi模块供应商
```

**场景2：项目资源调配**
```
邮件：G20项目进度滞后，需要人力支持

AI分析：
1. 查询G20项目 → 研发部负责
2. 查询部门信息 → 研发部负责人陈琦
3. 查询KPI指标 → 项目按时交付率是关键指标
4. 给出建议：
   - 通知研发部负责人陈琦
   - 考虑从其他项目临时调人
   - 预警：可能影响部门KPI
```

**场景3：客户满意度预警**
```
邮件：客户肖正伟连续3天反馈问题未解决

AI分析：
1. 查询客户信息 → 肖正伟是九胜科技CTO（重要客户）
2. 查询产品线 → G20项目
3. 查询部门KPI → 技术部的客户满意度KPI
4. 查询协作关系 → 技术部应联系研发部解决
5. 给出建议：
   - 高优先级警报
   - 通知技术部负责人徐智慧
   - 建议立即组织技术部+研发部联合会议
```

## 🔄 数据更新

### 更新频率建议
- **公司概况**：每季度或每年更新
- **业务领域**：每半年更新
- **产品线**：每次新产品发布时更新
- **部门职能**：组织架构调整时更新
- **部门协作关系**：流程优化时更新
- **业务流程**：流程变更时更新

### 更新方式
1. 直接编辑Excel文件
2. 保存后，程序会在下次运行时自动加载新数据
3. 无需重启程序

## ⚠️ 注意事项

1. **数据准确性**：确保部门信息与实际组织架构一致
2. **及时更新**：组织架构变化时及时更新
3. **权限管理**：敏感信息（如KPI目标）可选择性填写
4. **备份**：定期备份配置文件
