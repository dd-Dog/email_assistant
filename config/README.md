# Config 统一配置目录

## 📁 目录结构

```
config/
├── 公司/          ← 公司相关配置（V6.0新增）
├── 客户/          ← 客户相关配置（V6.0新增）
├── 供应商/        ← 供应商相关配置（V6.0新增）
└── 系统/          ← 系统配置（V6.0新增）
```

## 🎯 设计理念

### V6.0 渐进式迁移策略

**阶段1（当前）：V6.0与V5.3并存**
```
V6.0新配置：
└── config/
    ├── 公司/      ← 新增，公司知识图谱
    ├── 客户/      ← 新增，客户信息管理
    ├── 供应商/    ← 新增，供应商信息管理
    └── 系统/      ← 新增，系统级配置

V5.3旧配置（保留，继续使用）：
├── persons/       ← 人员信息（V5.3）
├── projects/      ← 项目文档（V5.3）
├── keyword_config.json
└── product_params.json

程序自动检测并兼容两种结构
```

**阶段2（V6.0稳定后）：统一迁移**
```
运行迁移工具：
python migrate_to_unified_config.py

迁移后：
└── config/
    ├── 人员/      ← 从 persons/ 迁移
    ├── 公司/      ← V6.0新增
    ├── 项目/      ← 从 projects/ 迁移
    ├── 客户/      ← V6.0新增
    ├── 供应商/    ← V6.0新增
    └── 系统/      ← 从 *.json 迁移
```

## 📂 各目录说明

### config/公司/
**用途**：存储公司整体信息，构建"公司知识图谱"

**文件**：
- `公司信息.xlsx` - 公司简介、业务范围、核心竞争力
- `部门信息.xlsx` - 部门职能、协作关系、KPI

**作用**：
- 让AI理解公司整体业务
- 为宏观分析提供基础
- 辅助决策建议生成

### config/客户/
**用途**：客户信息管理（未来与 persons/ 中的客户整合）

**文件**：
- `客户信息.xlsx` - 客户公司信息、行业、需求特点

**作用**：
- 客户关系管理
- 客户需求分析
- 客户满意度预警

### config/供应商/
**用途**：供应商信息管理（未来与 persons/ 中的供应商整合）

**文件**：
- `供应商信息.xlsx` - 供应商信息、产品线、合作历史

**作用**：
- 供应链管理
- 供货风险预警
- 价格趋势分析

### config/系统/
**用途**：系统级配置（未来从 JSON 迁移到 Excel）

**规划文件**：
- `关键词配置.xlsx` - 从 keyword_config.json 迁移
- `产品参数.xlsx` - 从 product_params.json 迁移
- `推送配置.xlsx` - 移动端推送规则
- `回复策略.xlsx` - 自动回复策略

### 程序自动检测逻辑

```python

if os.path.exists('config/公司'):
    logger.info("✅ 使用V6.0统一配置")
    company_mgr = CompanyManager('config/公司')

# 兼容V5.3旧结构
if os.path.exists('persons'):
    logger.info("✅ 使用V5.3人员配置（兼容模式）")

```

### V5.3与V6.0共存

在V6.0稳定之前：
- V5.3功能继续使用 `persons/`、`projects/` 等旧目录
- V6.0新功能使用 `config/` 新目录
- 两者互不干扰，平滑过渡
