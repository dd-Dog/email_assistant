## 🚀 快速开始

### 第一步：创建项目文件夹

为每个项目创建独立文件夹：

```bash
mkdir projects\G20
mkdir projects\889
mkdir projects\8088
```

### 第二步：添加项目文档

在项目文件夹中放入文档（支持格式）：

**支持的格式：**
- ✅ `.txt` - 纯文本
- ✅ `.md` / `.markdown` - Markdown
- ⏳ `.docx` - Word（下一版本）
- ⏳ `.xlsx` - Excel（下一版本）

**示例：`projects/G20/产品规格书.md`**
```markdown
# G20智能终端产品规格书

## 硬件规格
- 芯片：展锐T310
- 内存：3GB
- 存储：32GB
...

## 当前问题
1. 网络切换稳定性
2. 功耗优化
...
```

**AI会读取这些文档并用于分析！**

### 第三步：配置关键词

编辑 `keyword_config.json`：

```json
{
  "common_keywords": {
    "hardware": ["芯片", "T310", "展锐", "处理器", "内存"],
    "software": ["Android", "系统", "驱动", "应用"],
    "network": ["4G", "5G", "WiFi", "蓝牙", "GPS"],
    "bug": ["Bug", "问题", "故障", "异常", "崩溃"]
  },
  "project_keywords": {
    "G20": ["视频通话", "高清录音", "回声消除"],
    "889": ["5G", "WiFi6", "低功耗"]
  }
}
```

**作用：**
- `common_keywords` - 通用产品关键词（所有项目）
- `project_keywords` - 项目特定关键词

**效果：** 邮件中提到关键词 → 自动标记为重点邮件

### 第四步：配置产品参数

编辑 `product_params.json`：

```json
{
  "products": {
    "G20": {
      "hardware": {
        "chip": "展锐T310",
        "ram": "3GB LPDDR4X",
        "rom": "32GB eMMC"
      },
      "software": {
        "os": "Android",
        "android_version": "11"
      }
    }
  }
}
```

**作用：** AI知道产品参数，给出更精准的技术建议

### 第五步：运行测试

```bash
python main_v4.py
```

## 🎯 最佳实践

### 1. 项目文档组织

**推荐结构：**
```
projects/G20/
├── 产品规格书.md          ← 详细技术参数
├── 测试报告.txt           ← 测试进展和问题
├── 客户需求.md            ← 客户原始需求
├── 技术方案.md            ← 技术实现方案
└── 常见问题.txt           ← FAQ
```

**文档内容建议：**
- 产品规格：硬件参数、软件配置、功能列表
- 测试报告：当前进展、已知问题、负责人
- 客户需求：客户关注点、特殊要求
- 技术方案：实现思路、技术选型
- 常见问题：客户常问的问题和答案

### 2. 关键词配置

**通用关键词（common_keywords）：**
- 硬件：芯片型号、主要部件
- 软件：系统版本、核心功能
- 网络：连接方式
- 问题：Bug、故障相关词

**项目关键词（project_keywords）：**
- 项目特色功能
- 客户特别关注的点
- 项目特有的技术词汇

### 3. 产品参数

**只配置关键参数：**
```json
{
  "G20": {
    "hardware": {
      "chip": "展锐T310",  ← 最重要的
      "ram": "3GB",
      "rom": "32GB"
    }
  }
}
```

不需要所有参数都列出，AI会自动从项目文档中获取详细信息！

### 4. 持续更新

**每周更新：**
- 测试报告：最新进展
- 已知问题：新发现的问题

**每月更新：**
- 产品规格：参数变更
- 技术方案：方案调整

## 🔧 配置文件说明

### keyword_config.json

```json
{
  "common_keywords": {
    "分类名": ["关键词1", "关键词2", ...]
  },
  "project_keywords": {
    "项目代码": ["关键词1", "关键词2", ...]
  }
}
```

**分类建议：**
- `hardware` - 硬件相关
- `software` - 软件相关
- `network` - 网络相关
- `performance` - 性能相关
- `quality` - 质量相关
- `bug` - 问题相关
- `feature` - 功能相关

### product_params.json

```json
{
  "common_params": {
    "分类": {
      "参数key": {"name": "显示名", "type": "类型"}
    }
  },
  "products": {
    "产品代码": {
      "分类": {
        "参数key": "参数值"
      }
    }
  }
}
```
