## 🆕 三大核心功能

### 1️⃣ 项目文档加载器 📁

**问题：** V5.0的项目信息只能在JSON中配置，内容有限

**解决：** 从文件夹加载完整项目文档！

```
projects/
├── G20/
│   ├── 产品规格书.md      ← 详细硬件软件参数
│   ├── 测试报告.txt        ← 当前问题和进展
│   └── 客户需求.md        ← 客户要求
├── 889/
│   └── 项目简介.txt
└── 8088/
    └── ...
```

**支持格式：**
- ✅ TXT - 纯文本
- ✅ MD / Markdown - Markdown格式
- ⏳ DOCX / XLSX - 下一版本

**效果：**
- AI可以读取所有项目文档
- 项目信息更全面、更准确
- 不需要手动整理到JSON

### 2️⃣ 关键词分类系统 🏷️

**问题：** 无法自动识别重点邮件

**解决：** 双层关键词系统！

```json
{
  "common_keywords": {
    "hardware": ["芯片", "T310", "展锐"],
    "bug": ["Bug", "故障", "崩溃"]
  },
  "project_keywords": {
    "G20": ["视频通话", "回声消除"],
    "889": ["5G", "WiFi6"]
  }
}
```

**效果：**
```
2025-11-11 14:30 G20视频通话中断 
[🔴紧急] [项目:G20] [network:WiFi,4G | bug:中断]  ← 自动标记
```

**价值：**
- 重点邮件一眼识别
- 涉及关键词自动标记
- AI重点关注这些内容

### 3️⃣ 产品参数管理系统 📊

**问题：** 产品参数上百项，无法有效管理

**解决：** 结构化参数系统！

```json
{
  "products": {
    "G20": {
      "hardware": {
        "chip": "展锐T310",
        "ram": "3GB LPDDR4X"
      },
      "software": {
        "android_version": "11"
      }
    }
  }
}
```

**效果：**
- AI知道产品参数
- 给建议时考虑实际硬件能力
- 参数可搜索、可格式化

## 📊 实际效果对比

### 场景：客户反馈网络问题

**邮件：** "G20从4G切换WiFi时视频通话中断"

#### V5.0 分析
```
📝 客户报告网络切换问题
⚙️  可行性：需要优化网络切换逻辑
🔧 实现：检查网络状态 | 增加缓冲 | 实现无缝切换
💡 建议：先确认具体场景
```

#### V5.1 分析（基于知识库）
```
[🔴紧急] [项目:G20] [network:WiFi,4G | bug:中断]  ← 关键词标记

📋 G20智能终端 | 测试阶段 | 85%

📝 客户肖正伟（技术专家）报告网络切换问题

【项目文档摘要】
=== 测试报告.txt ===
中优先级问题：网络切换时视频通话中断
负责人：李越征
截止日期：2025-11-25

⚙️  可行性：已知问题。根据测试报告，团队正在处理中（李越征负责）

🔧 实现方案：
    - 参考测试报告中的优化方案
    - 优化网络切换逻辑（开发中）
    - 基于T310芯片和3GB内存实现缓冲
    - 预计11月25日完成

💡 建议：
    1. 告知客户这是已知问题，正在处理
    2. 预计11月25日解决
    3. 可提供临时解决方案（手动切换）
    4. 与李越征同步最新进展
    
✓ 今天回复客户说明进度 | 与李越征确认方案
```

**差异：**
- ✅ 关键词自动标记（重点邮件）
- ✅ 知道这是测试报告中的已知问题
- ✅ 知道负责人和截止时间
- ✅ 基于硬件参数给具体方案
- ✅ 建议可直接执行

## 💡 使用建议

### 快速开始（5步）

**Step 1：** 创建项目文件夹
```bash
mkdir projects\G20
mkdir projects\889
```

**Step 2：** 放入项目文档
```bash
# 把现有的Word/Excel文档转成TXT或MD
# 放到对应项目文件夹
projects\G20\产品规格书.md
projects\G20\测试报告.txt
```

**Step 3：** 配置关键词
```json
// keyword_config.json
{
  "common_keywords": {
    "bug": ["Bug", "故障", "异常"]
  },
  "project_keywords": {
    "G20": ["视频通话", "录音"]
  }
}
```

**Step 4：** 配置产品参数（可选）
```json
// product_params.json
{
  "products": {
    "G20": {
      "hardware": {"chip": "展锐T310"}
    }
  }
}
```

**Step 5：** 运行测试
```bash
python main_v4.py
```

## 🎓 适用场景

### 推荐使用V5.1的场景

✅ **多项目并行**
- 每个项目有大量文档
- 需要快速了解项目背景

✅ **产品参数复杂**
- 硬件软件参数上百项
- 客户经常咨询参数

✅ **重点内容多**
- 需要快速识别重点邮件
- 某些关键词需要特别关注

✅ **团队协作**
- 多人参与项目
- 需要知道谁负责什么

### 继续使用V5.0的场景

✅ 项目简单，文档少  
✅ 不需要关键词标记  
✅ 追求极简配置
