## 🏢 客户需求AI分析（重点）

### 专门针对客户邮件的分析

AI会特别分析：
1. **📝 需求简述** - 客户想要什么（1-2句话）
2. **⚙️ 可行性评估** - 技术上能否实现
3. **🔧 实现方法** - 关键实现思路（2-3个要点）
4. **💡 简洁建议** - 应对建议（1-2句话）
5. **✓ 行动项** - 需要做什么
6. **📅 截止时间** - 如果提到了

### 报告示例

```
█ 🏢 客户需求邮件 (2封) - AI需求分析

▸ A客户公司 (client@customer.com) - 2封
  2025-11-11 14:30 需要增加视频录制功能 [🔴紧急]
    📝 需求: 客户要求在设备上增加本地视频录制功能
    ⚙️  可行性: 可行，需要增加存储模块和视频编码器
    🔧 实现: 1.评估存储空间需求 | 2.选择合适的视频编码方案 | 3.预计2-3周开发周期
    💡 建议: 先与客户确认视频质量要求和存储时长，再给出详细方案
    ✓ 今天回复客户 | 周五前提供技术方案
    📅 本周五
  
  2025-11-10 09:15 设备启动慢的问题 [🟡普通]
    📝 需求: 客户反馈设备启动时间过长，影响使用
    ⚙️  可行性: 可优化，需要分析启动流程
    🔧 实现: 1.profiling启动流程 | 2.优化启动加载项 | 3.可能1周完成
    💡 建议: 先获取设备日志，分析具体慢在哪个环节
    ✓ 联系客户获取日志 | 分析启动瓶颈
```

## 🔌 供应商邮件分析

AI会分析：
1. **简述** - 邮件主要内容
2. **是否需要回复** - 提醒及时响应
3. **关键信息** - 重要时间节点、技术细节等
4. **行动项** - 需要做什么

### 报告示例

```
█ 🔌 供应商邮件汇总 (1封)

▸ 高通芯片 (contact@chipmaker.com) - 1封
  2025-11-11 16:20 新固件版本发布通知 [🟡普通]
    💡 供应商发布了新固件v2.1.5，修复了3个已知bug
    ✓ 下载新固件 | 安排测试验证
```

## 🔧 配置技巧

### 1. 客户配置
- 使用公司名作为姓名：`"客户A公司"` 而不是 `"张三"`
- 便于快速识别客户来源

### 2. 供应商配置
- 使用供应商品牌名：`"高通"`、`"联发科"`
- 清晰明了

### 3. 空配置也OK
```json
{
  "customers": {},    // 暂时没有客户邮件也可以
  "suppliers": {}     // 留空不影响使用
}
```
