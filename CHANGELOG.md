# 更新日志

## V2.0 - 2025-11-11

### 🎉 重大更新

1. **关键人配置优化**
   - 分离了领导和员工两种配置
   - 使用 key-value 格式：`{"邮箱地址": "姓名"}`
   - 显示格式更直观：`姓名(邮箱地址)`

2. **邮件分类展示**
   - 领导邮件：全部内容展示，按发件人分组
   - 员工邮件：全部内容展示，按发件人分组
   - 重复问题单独显示在最前面

3. **智能重复问题检测**
   - 只对员工邮件进行重复问题检测
   - 领导邮件始终完整展示
   - 连续3天类似内容会被标记为重复问题

4. **报告优化**
   - 新的视觉设计，领导邮件和员工邮件区分显示
   - 重复问题优先显示，更加醒目
   - 按发件人分组，查看更清晰

### 📝 配置文件变更

**旧版配置（V1.0）：**
```json
{
  "key_senders": [
    "sender1@example.com",
    "sender2@example.com"
  ]
}
```

**新版配置（V2.0）：**
```json
{
  "leaders": {
    "boss@company.com": "老板",
    "manager@company.com": "经理"
  },
  "employees": {
    "colleague1@company.com": "张三",
    "colleague2@company.com": "李四"
  }
}
```

### 🔄 升级指南

如果您从 V1.0 升级到 V2.0，需要修改 `config.json`：

1. 将 `key_senders` 数组改为 `leaders` 和 `employees` 对象
2. 为每个邮箱地址配置对应的姓名
3. 参考 `config.example.json` 的新格式

### ⚙️ 技术改进

- 优化了邮件数据结构，增加了 `from_email` 和 `from_name` 字段
- 改进了分析器，支持按发件人分组
- 重构了报告生成器，支持分类展示
- 更新了主程序逻辑，分别处理两类邮件
